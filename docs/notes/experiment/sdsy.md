## 课程概述

>分数构成：平时70%（到课 + 实验完成情况 + 报告） + 上机考试30%

数电实验是微电子和信工的大二必修课，和数电理论课的内容其实有很多是对应的（如常见的组合逻辑模块及其Verilog描述），二者结合起来可以更好的理解数电 *（插一嘴，对数电感兴趣的还可以去玩《图灵完备》(Turing Complete)）*

与其他实验课不太一样，数电实验采用自主完成lab的形式，平时自己写代码为主、老师讲课为辅；这种形式其实挺好的，可以锻炼<del>我们的</del>ai的debug能力，而且老师答疑也非常耐心，常常能一眼指出问题所在（*果然硬件越老越妖(bushi)*）；只不过我自己比较喜欢闷声钻研，和老师的交流比较少:sob:

今年考试是两道题，一道控制器设计题（会给ASM或状态转移图），一道代码改错题（语法错误、逻辑错误）；经验是先做设计题，因为改错题很难把错误都找全，还要花大量时间...

另外，在vibe coding 越来越完备的浪潮下，<del>我这学期的代码已经几乎没有自己手写过了（）</del>，可能数电实验这些课难度越来越低了，据说之后也要进行课程改革了；但我认为，关键不在于手不手写代码（aka“古法编程”），而在对代码的理解和应用，毕竟代码和语法就算记得再牢也总有一天会忘记了，掌握了方法才能在合适的时候想起来怎么用。

另外，老师强调的文件管理，也确实是非常重要的...
